## Architecture Overview

Pattern: Static Single Page Application (SPA) with pre-rendered routes.
Framework: SvelteKit 5 with adapter-static — all routes are pre-rendered at build time.
State Management: Svelte writable store (courseStore) acting as a singleton — persists to localStorage on every mutation.
Rendering: Client-side only after hydration. No SSR. Layout is shared across all routes via +layout.svelte.
Routing: File-based routing via SvelteKit. Key routes: / (home/module list), /modulo/[id] (module detail), /resultados (results/certificate).
Data Flow: modules.ts (static data) → +page.svelte (render) ↔ courseStore (state) ↔ localStorage (persistence).
Deployment: GitHub Actions → npm ci → vite build (BASE_PATH=/go-course) → GitHub Pages.
